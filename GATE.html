Form Element and Attributes

"The <form> HTML element represents a document section containing interactive controls for submitting information"
    - MDN

    Example: <form></form>

  • Elements: The building blocks of code.
  • Attributes: Additional values that can provide information about the elements.

The <action> attribute indicates where form data should be sent.

    Example: <form action="/submit-url"> </form>



Input Element and Attributes

The <input> element allows you several ways to collect data from a web form.

    Example: <input> (Input elements are self-closing, so there is no need to add a closing tag) Input elements are nested inside <form> elements.

There are many kinds of inputs you can create using the <type> attribute. You can easily create a password field, reset button, or a control to let users select a file from their computer.

    Example: <input type="text">

In order for a form's data to be accessed by the location specified in the <action> attribute, you must give the text field a <name> attribute and assign it a value to represent the data being submitted. For example, you can use the following syntax for an email address text field.

    Example: <input type="text" name="email">

Placeholder text is used to give people a hint about what kind of information to enter into an input.

    Example: <input type="text" placeholder="email address">

To prevent a user from submitting your form when required information is mssing, you need to add the <required> attribute to an <input> element. There's no need to set a value to the <required> attribute.



Button Element and Attributes

Use the <button> element to create clickable buttons.

    Example: <button> Click Here </button> creates the button with the text "Click Here."

Note

The default behavior of clicking a form button without any attributes submits the form to the location specified in the form's <action> attribute.


Note

Both <input> and <button> elements are "inline" elements, which means they don't appear on new lines.

Although a button will submit by default, relying on this can cause confusion. When using a <button> element, add the <type> attribute with value "submit" to the <button> to make it clear that this is a submit button.

You can use radio buttons for questions where you want only one answer out of multiple options. Example of radio button with the option of "cat."

    Example: <input type="radio"> Cat

Note

The radio button needs to be written as a new element. Ensure not to add this to the <input> element associated with the "submission" box.



Label Element and Attributes

<Label> elements are used to help associate the text for an <input> element with the <input> element itself (especially for assitive technologies like screen reader).

    Example: <label><input type="radio">cat</label> In this example, clicking on the text "cat" will also select the corresponding radio button

The <id> attribute is used to identify specific HTML elements. Each <id> attribute's value must be unique from all other <id> values for the entire page. When elements have multiple attributes, the order of the attributes doesn't matter.

When there are multiple radio buttons to choose from, you can by default, select more than one. To solve this, add the <name> attribute to both buttons, and give it the same value for both buttons. When this is done, selecting one button will automatically deselect the other.

If you select one of the radio buttons, and submit the form, the form data for the button is based on it's <name> and <value> attributes. Since your radio buttons do not have a <value> attribute the form data will include, for example, <indoor-outdoor=on>, which is not useful when you have multiple buttons. To fix this, add a <value> attribute to the radio buttons. For convenience, set the button's <value> attribute to the same value as its <id> attribute.



Fieldset Element and Attributes

The <fieldset> element is used to group related inputs and labels together in a web form. <fieldset> elements are block-level elements, meaning that they appear on a new line, versus in-line elements, meaning that they appear on the same line.



Legend Element and Attributes

The <legend> acts as a caption for the context in the <fieldset> element. It gives users context about what they should enter into that part of the form.

Unlike the <radio> attribute for the <input> element, which only allows one option to be selected, checkboxes are commonly used in forms for questions that may have more than one answer. For example, here's a checkbox with the option of "tacos:"

    Example: <input type="checkbox"> tacos

There's another way to associate an <input> element's text with the element itself. You can nest the text within a <label> element and add a <for> attribute with the same value as the <input> element's <id> attribute. When doing this, nest only the <input> element's text in a <label> element.

Note Like radio buttons, form data for selected checkboxes are <name> / <value> attribute pairs. While the <value> attribute is optional, it's best practice to include it with any ceckboxes or radio buttons on the page.

In order to make a checkbox or radio button selected by default, you need to add the <checked> attribute to it. There's no need to set a value to the <checked> attribute. Instead, just add the word "checked" to the <input> element, making sure there is space between it and other attributes.
